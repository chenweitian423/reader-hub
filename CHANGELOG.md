# Changelog

## 1.19.1 - 2026-04-24

- 修复一类站点“自动识别后测试搜索直接 404”的问题。
- 自动识别现在会优先探测页面里的搜索表单、搜索链接和常见搜索路径，尽量回填真实可访问的搜索入口，而不再盲猜固定的 `/search`。

## 1.19.0 - 2026-04-24

- 私有站点接入器新增“小说网址自动识别”入口，粘贴书籍详情页或章节页网址后，系统会自动猜测站点类型并尽量回填规则。
- 自动识别现在会尝试推断站点首页、搜索表单、详情页元数据、目录列表、目录翻页、正文区域和正文翻页规则。
- 自动识别结果会直接回填到表单，并在预览区展示本次猜测的提示信息，方便继续测试和微调。

## 1.18.0 - 2026-04-24

- 私有站点接入器新增可视化高级设置，详情页元数据规则、目录下一页规则和正文下一页规则现在都可以直接在表单里配置。
- 新增五类常见站点模板，覆盖 PC HTML 列表站、移动站多页正文、JSON API 站、JSON 搜索 + HTML 阅读，以及 XPath 站点。
- 新增规则速查卡片和更丰富的测试预览标签，能直接看出当前站点是否已配置详情元数据、目录翻页和正文翻页能力。

## 1.17.0 - 2026-04-24

- 书源管理新增“私有站点接入”卡片，支持通过表单填写搜索、目录和正文规则，先测试再导入书源。
- 后端新增私有站点预览、测试和导入接口，适合接入你自己维护或明确授权的小说站点。
- 站点测试结果现在会直接在页面里展示命中书籍预览，并标记当前规则是否已经具备阅读能力。

## 1.16.0 - 2026-04-24

- 书源管理新增多选删除能力，支持本页全选、清空选择和删除所选书源。
- 新增“删除全部”动作，可一键清空全部可删除书源，系统内置的“本地导入书库”会自动保留。
- 书源列表卡片新增勾选列与批量操作区，删除类操作的禁用态和提示文案也一并补齐。

## 1.15.1 - 2026-04-24

- 修复前端静态资源缓存导致的“新页面结构配旧脚本”问题。
- 首页现在会为 `favicon`、`styles.css`、`app.js` 自动追加版本号查询参数，版本更新后浏览器会更可靠地拉取新资源。
- 连带修复书源管理分页控件显示了但仍按旧逻辑渲染全部书源的问题。

## 1.15.0 - 2026-04-24

- 把“当前设备导入”和“局域网设备上传”在界面与数据展示上彻底拆开，避免继续给人“重复功能”的感觉。
- 书架页新增两块独立入口：本机导入区保留拖拽与目录选择，局域网设备入口单独展示上传页链接。
- 本地导入书和局域网上传书现在会保存不同的导入来源标记，并在书架与详情里显示。
- 当前设备导入与局域网上传页都新增了上传进度条。

## 1.14.2 - 2026-04-24

- 调整 `/api/library/uploads` 的浏览器访问行为：现在直接在浏览器打开这个地址，会返回一个可直接选择文件、选择目录、拖拽上传的网页，而不再只显示 JSON 接口说明。
- 保留原有接口能力，非浏览器或非 HTML 请求仍然可以继续拿到 JSON 说明和 POST 上传接口。

## 1.14.1 - 2026-04-24

- 继续增强书架页上传交互：新增浏览器原生文件选择和目录选择能力，并保留传统 input 回退，提升桌面浏览器兼容性。
- 新增全页拖拽防误触处理，拖文件到页面其他位置时不再容易把当前页面直接替换成文件内容。
- 书源管理新增分页和每页数量控制，大量书源时不再一屏拉得过长；书源说明默认压成两行，页面更紧凑。

## 1.14.0 - 2026-04-24

- 重做书架页本地导入区域，新增独立投放面板，支持点击选择文件、点击选择整个目录，以及直接拖动文件或文件夹上传。
- 新增待上传文件队列与文件清单展示，上传前可以先确认来源目录、文件数量和体积，再一键导入到书架。
- 拖动上传现在会自动递归读取目录内容，并过滤不支持的格式，只保留 TXT、MD、EPUB。

## 1.13.1 - 2026-04-24

- 修复局域网上传接口体验问题：现在直接访问 `/api/library/uploads` 会返回接口说明，不再只给出 405。
- 新增跨域支持，方便同一网络下的其他网页、小工具或表单直接调用上传接口。

## 1.13.0 - 2026-04-23

- 新增本地书籍导入功能，支持在书架页直接导入 TXT、MD、EPUB 文件到书架。
- 新增局域网书籍上传接口，同一网络下的设备可以直接通过 `/api/library/uploads` 上传书籍进入书架。
- 新增“本地导入书库”内置书源，导入的本地书籍现在也能参与搜索、打开目录和阅读正文。

## 1.12.5 - 2026-04-23

- 修复一类旧格式书源“能导入、能搜索，但加入书架后无法阅读”的问题。
- 后端新增旧格式书源的目录页与正文页解析，常见 `ruleToc` / `ruleContent` 规则现在可以直接打开章节阅读。
- 前端对书源“支持阅读”的判断同步修正，旧格式可读书源不再被误判成“仅搜索”。

## 1.12.4 - 2026-04-23

- 继续简化首页“搜索状态”区域，改成更轻量的摘要条与统计胶囊，减少大块状态卡对首页的干扰。
- 命中来源改为紧凑标签展示，点标签可直接切换到该书源结果。
- 异常来源与零命中来源统一收进折叠区，让首页在信息完整的同时保持更干净。

## 1.12.3 - 2026-04-23

- 重做首页搜索结果区：搜索完成后会自动滚动到结果区，并明确展示“左侧可选书单 + 右侧详情”的选书结构。
- 首页书源状态改成摘要视图，优先突出命中来源与失败来源，把“命中 0 本”的书源折叠起来，避免抢占页面视觉焦点。
- 优化搜索结果卡片层级与选中反馈，让搜出书后更容易发现“可以点哪一本继续看”。

## 1.12.2 - 2026-04-23

- 优化书源列表和搜索状态的换行与布局，避免长格式说明或错误信息把页面撑坏。
- 修复已导入书源后搜索空态仍提示“先去导入书源”的误导文案。
- 优化搜索结果反馈：现在会明确区分“无结果”“部分来源失败”“全部来源失败”。

## 1.12.1 - 2026-04-23

- 放宽旧格式书源的导入校验，更多带复杂规则的旧格式书源现在可以先成功导入。
- 对暂不支持的旧格式规则，改为在搜索阶段按来源单独报错，不再卡死在导入阶段。

## 1.12.0 - 2026-04-23

- 修复导入不兼容书源格式时后端直接抛出 500 的问题，现在会返回明确的 400 错误信息。
- 新增对一类常见旧格式书源的兼容导入与搜索能力，现已支持 JSON 规则以及常见 HTML/CSS、XPath 搜索规则。
- 当旧格式规则使用 JS 等暂不支持语法时，接口会给出清晰提示，方便判断是否需要换源。

## 1.11.0 - 2026-04-23

- 独立阅读页新增“目录抽屉”，即使收起侧栏也能快速切换章节。
- 新增底部翻页条，阅读时可以更顺手地查看当前章节并切换上一章、下一章。
- 阅读页顶部标题区和工具栏会随滚动自动收起，正文区域更沉浸。

## 1.10.0 - 2026-04-23

- 阅读器从搜书首页底部拆成独立阅读页，打开书籍后会进入更宽、更完整的阅读界面。
- 独立阅读页新增“返回上一页”“去书架”和“收起侧栏”操作，让目录、设置与正文切换更顺手。
- 优化正文区域宽度和高度，阅读页整体更适合长时间阅读。

## 1.9.0 - 2026-04-23

- 首页新增“最近搜索”“最近更新”“相关推荐”三个书城式发现分区，让搜索页更像可逛的书城首页。
- 搜索结果详情面板补充了关键信息卡片与同作者/同来源推荐，帮助更快完成选书决策。
- 阅读器新增章节进度条与“沉浸模式”，让正文阅读更接近正式读书产品体验。

## 1.8.0 - 2026-04-23

- 首页新增“搜索结果详情面板”，点击结果卡片即可在右侧查看更完整的书籍信息与操作按钮。
- 首页新增更像书城分区的“可立即阅读”和“已在书架”内容带，提升浏览感和决策效率。
- 搜索结果列表增加选中态，首页信息架构继续向正式产品靠拢。

## 1.7.0 - 2026-04-23

- 首页继续朝正式读书产品演进，新增“焦点书籍”和“继续阅读”模块。
- 首页现在会根据当前阅读、搜索结果或书架内容，自动展示更适合打开的一本书。
- 新增最近在读卡片区，支持从首页直接续读书架里的书。
- 优化首页视觉层次，让搜书页更像书城首页而不是管理后台。

## 1.6.0 - 2026-04-23

- 重做前端信息架构：首页现在只保留搜书、搜索结果和阅读器。
- 新增左侧菜单导航，把书架、书源管理、数据工具拆成独立页面，不再挤在同一屏。
- 优化整体视觉样式，增强菜单层级、首页聚焦感和移动端布局表现。
- 首页新增快捷搜索词按钮，书架继续阅读会自动切回搜书首页。

## 1.5.1 - 2026-04-23

- 修复 Docker 镜像未包含 `VERSION` 文件导致服务接口返回版本号 `0.0.0` 的问题。
- 发布镜像和服务器部署现在会正确显示应用版本号。

## 1.5.0 - 2026-04-23

- 新增应用版本文件 `VERSION`，后端、文档和 Docker 发布流程统一读取版本号。
- 新增 `CHANGELOG.md`，每次迭代都可以追加版本更新说明。
- 新增数据备份与恢复能力，支持导出书源、书架、缓存章节和阅读设置，并支持合并导入或覆盖恢复。
- Docker Hub 镜像发布新增版本标签，除了 `latest` 和 `sha-*` 以外，还会生成 `1.5.0` 与 `1.5` 这样的版本 tag。
- `docker-compose.yml` 支持通过 `READER_HUB_IMAGE_TAG` 固定部署指定镜像版本。
